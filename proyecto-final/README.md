# Proyecto Final

El proyecto final es un análisis de datos completo, hecho de forma individual, sobre un dataset de tu elección.

---

## ¿Qué tenés que entregar?

Un repositorio en GitHub que incluya:

1. **README.md** — Descripción del proyecto, el problema que analizaste y los insights principales
2. **notebook.ipynb** — Análisis completo y documentado
3. **dataset** — El archivo de datos usado (o link a la fuente si es muy grande)
4. **presentación** — Slides o documento con los resultados para la Demo Week

---

## Etapas del proyecto

### 1. Elegí tu dataset (Clase 18)
Elegí un dataset que te genere una pregunta real. Tiene que tener al menos 500 filas y múltiples columnas.

Ejemplos de problemas válidos:
- ¿En qué zona de Buenos Aires hay más entregas demoradas?
- ¿Qué tipo de producto tiene mejor conversión en este e-commerce?
- ¿Cómo cambió el comportamiento de usuarios entre enero y junio?

### 2. Limpieza y exploración (Semana 10–11)
- Tratá los valores nulos y duplicados
- Explorá distribuciones y detectá outliers
- Documentá cada decisión que tomás

### 3. Análisis (Semana 11)
- Respondé la pregunta con datos
- Calculá las métricas relevantes
- Encontrá al menos 3 insights accionables

### 4. Visualización (Semana 11–12)
- Al menos 4 gráficos relevantes
- Cada gráfico debe tener título, labels y una conclusión en el notebook

### 5. Presentación — Demo Week (Clase 24)
- 10 minutos de presentación
- 5 minutos de preguntas del grupo
- Enfocate en los insights, no en el código

## Criterios de evaluación

| Criterio | Peso |
|----------|------|
| Calidad del análisis y profundidad | 30% |
| Limpieza y documentación del código | 20% |
| Calidad de las visualizaciones | 20% |
| Claridad de los insights presentados | 20% |
| README del repositorio | 10% |
